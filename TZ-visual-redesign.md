# ТЗ: Візуальне підсилення + технічні SEO-фікси

Базується на аудиті від 2026-07-06. Мета — усунути враження "занадто простого"
сайту, не змінюючи кольорову ідентичність (амбер/тіл "signal pipeline"), і
закрити технічні SEO-гепи знайдені в коді.

## 1. Технічні фікси (quick wins)

- **Title/description**: скоротити до ліміту показу в Google (title ≤60 симв.,
  description ≤160 симв.) в `index.html` та `en/index.html`.
- **Favicon**: SVG-іконка на основі бренд-крапки (`●`, колір `--signal`),
  підключена через `<link rel="icon">` в обох файлах.
- **og:image**: згенерувати статичне зображення 1200×630 (бренд-кольори,
  ім'я + позиціонування), додати `og:image` / `twitter:image` в `<head>`.
- **Тап-таргети**: `.socials a` з 38px → 44px (мобільна доступність).

## 2. Дизайн-система "візуальної насиченості"

Переносимо принцип з `tailwind-preview.html` (icon-badge, stat-box,
floating-badge, project-visual, feature-badge), але в амбер/тіл палітрі,
без Tailwind CDN — чистий CSS у `assets/style.css`, щоб не плодити другий
фреймворк у проді.

**Нові CSS-компоненти:**
- `.icon-badge` + 4 кольорові варіанти (`--signal` амбер, `--wire` тіл,
  фіолетовий і зелений акценти для різноманіття) — квадратна іконка з
  обведенням, використовується у Values / Skills / Services.
- `.stat-box` — невеликий рядок "цифр" без вигаданих показників: "5+ років
  досвіду", "Відповідь протягом дня", "UA · Remote" — чесні, вже наявні дані,
  просто винесені у візуальний блок біля hero.
- `.floating-badge` — плаваючий бейдж біля фото в hero: "Відповідаю
  протягом дня" / "Available today".
- `.project-visual` — верхній блок картки проєкту: іконка на кольоровому
  градієнті (без зовнішніх фото — уникаємо залежності від хотлінків і
  проблем з ліцензіями, які вже виникали з Unsplash+).
- `.feature-badge` — рядок у футері: "5+ років досвіду", "Відповідь за
  день", "Україна · Remote".

## 3. Зачіпані файли

- `index.html`, `en/index.html` — розмітка (іконки, badge, project-visual)
- `assets/style.css` — нові класи
- `assets/favicon.svg` (новий)
- `assets/og-image.png` (новий, згенерований)

## 4. Явно поза скоупом (без змін)

- Домен (`example.com`), контакти (`@your_telegram_username`,
  `hello@example.com`) — чекають реальних даних.
- Деплой Cloudflare Worker — інструкція вже готова в `worker/README.md`,
  виконує користувач.
- Реальні відгуки клієнтів — заглушка залишається до перших проєктів.
- Фото в проєктах — свідомо замінені на іконки+градієнт замість фото, щоб
  не залежати від зовнішніх стокових картинок; можна замінити на реальні
  скріншоти проєктів пізніше.

## 5. Критерії приймання

- Обидві мовні версії візуально ідентичні за структурою.
- Жоден існуючий функціонал (форма, nav, reveal-анімації) не зламаний.
- Lighthouse-подібна перевірка вручну: favicon є, og:image віддається,
  title/description в межах лімітів.
- Скріншот до/після для порівняння.
